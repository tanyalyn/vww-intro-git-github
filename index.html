What is ?

Git is an open-source, distributed version control system. It:

  • tracks changes to documents
  • makes collaboration easy
  • keeps copies on every users’s computer
  • makes recovering previous changes possible
  • helps with worry-free experimenting
  • and more…

What is ?

GitHub is a web-based host for Git repositories and:

  • allows users to collaborate
  • makes open-source code changes simple
  • has a built-in bug tracker
  • hosts simple websites from your repositories
  • and much, much more…

Git Basics: reset

The status after committing—README.md still contains changes, but hasn't been committed.

GUI

Git Basics: reset

In the CLI, add only the files you want to commit. If you add a file you don’t want to commit, use reset to unstage the file.

CLI

hello-world$  git add README.md
hello-world$  git add my-image.png
hello-world$  git reset HEAD README.md
Unstaged changes after reset:
M	README.md
hello-world$

Git Basics: reset

The status after reset. Note that status provides helpful reminders for how to add or remove files from the staging area.

CLI

hello-world$  git status
On branch master
Changes to be committed:
  (use "git reset HEAD ..." to unstage)

	new file:   my-image.png

Changes not staged for commit:
  (use "git add ..." to update what will be committed)
  (use "git checkout -- ..." to discard changes in working directory)

	modified:   README.md
hello-world$

Git Basics: Branching

Branches make it simple to work on experiments, major content changes, or bug fixes without danger of messing up existing work. You can continue to commit, but it won't change your main code base.
